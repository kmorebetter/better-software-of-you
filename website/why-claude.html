05 · The platform

Why it has to be
Claude Code specifically.

SoY is a Claude Code plugin — not a browser extension, not a separate app, not a Chrome sidebar. That choice is deliberate.

{ }
Structured data, not pasted text

Your context isn't dumped into a chat window as raw text. SoY exposes it as structured, typed data — the way Claude reads files. Relationships have schemas. Projects have fields. Decisions have metadata. Claude reads context the same way it reads code.

⌘
Where your most important work happens

Claude Code is not a chat interface — it's a development environment with context-aware tooling. When you use Claude Code for writing, analysis, planning, and execution, SoY is already there. No context-switching. No copy-paste ritual.

↗
Native tool use, not prompt stuffing

SoY registers as native Claude Code tools. Claude calls them when relevant — without you asking. It doesn't wait for you to type "here's some context." It reads what it needs, when it needs it, as a first-class participant in the session.

∞
Persistent across every session

Claude Code's plugin system persists configuration across sessions. SoY is loaded every time you open Claude Code. Not because you remembered to paste it — because it's part of the environment. Like a colleague who's always in the room.

plugin_type
Claude Code MCP (Model Context Protocol)
context_access
Structured tool calls — Claude requests context like reading a file, not scanning a paste
persistence
Session-permanent — loaded on every Claude Code launch, no setup required
data_storage
Local SQLite database — your context never leaves your machine unless you choose
activation
One install command — then SoY is live in every session automatically
