The Shift

Context engineering is
what comes after
prompt engineering.

What is context engineering?

"Building what your AI knows — not just how you ask it."

Prompt engineering optimises the question. Context engineering builds the world the AI operates in. The difference between asking a stranger for directions versus asking someone who's been living in your life for a year. Same question. Completely different answer.

Prompt Engineering

"Craft a better question and hope the model infers what you mean."

Every session starts fresh. Every gain expires. The ceiling is how much you remembered to paste in.

Context Engineering

"Build what the AI knows — and every future question gets smarter."

Compounds. Every contact you add, every conversation you log — each one makes every future interaction more precise. Not because the model improved. Because the context did.

Software of You is the context engineering platform for Claude Code. One SQLite database on your machine. Eight cross-referenced modules. 45+ slash commands. Claude is the interface.

Before You Buy

Common questions.

I already use Claude's built-in memory. Why do I need this? +
Claude's native memory stores summaries and preferences — useful for knowing your name and tone. SoY builds a structured, cross-referenced database: 45+ slash commands, contact relationship scores derived from email tone, meeting transcripts with extracted commitments, projects linked to the clients who drive them, decisions tracked against outcomes. Native memory gives Claude a sticky note. SoY gives it a filing system.
I don't use Claude Code that heavily. Is this still worth it? +
SoY compounds with use — but it doesn't require heavy use to be valuable. If you have meetings, clients, projects, or decisions worth remembering, SoY makes them available to Claude from the first query. The question isn't how often you use Claude Code today — it's how often you wished Claude knew more about your work. If that's more than once a week, SoY earns its place.
I'm not a developer. Can I use this? +
SoY runs inside Claude Code, so you need Claude Code installed. If you use Claude Code regularly, SoY installs in minutes. You interact entirely through natural language — no SQL, no terminal commands, no configuration. If Claude Code is part of your workflow, SoY fits inside it.
What happens to my data if I stop using it? +
Nothing. The database lives on your machine in standard SQLite format. SoY never had your data. You always did. Stop using the plugin and the database stays exactly where it is — readable, exportable, yours.
How is this different from a CRM like HubSpot? +
HubSpot is a team sales tool optimised for pipeline management. SoY is a personal context system. In HubSpot, a contact is a contact. In SoY, a contact is linked to every project you share, every conversation you've logged, every email in your thread, every decision they influenced. And all of it is queryable by Claude in natural language, inside the environment where you work.
What if product updates break my data? +
Auto-backup runs before every migration. If anything goes wrong during an update, the system auto-restores from the last snapshot. Your data has never been at risk from an update.
